# Candidate Future Domain Notes

## Overview

The following domains are identified as candidates for future application of the CPFC protocol. These domains are NOT empirically validated in this paper and are presented as conceptual extensions only.

## Important Disclaimer

All domains listed here are **candidate domains only**. They have not been subjected to the full CPFC protocol verification process described in this manuscript. Any application of the ordinal validation-entropy framework to these domains would require independent validation of P1-P4 conditions.

## Candidate Domains

### 1. Code Generation with Unit Tests
Code generation tasks where correctness is verified by unit test pass/fail outcomes. Discrete checkpointability comes from individual test assertions. External rules come from test specifications.

### 2. Formal Proof Fragments
Mathematical or logical proof steps verified against formal proof libraries. Each proof step can be independently checked by automated theorem provers.

### 3. Schema-Constrained QA
Question-answering tasks where answers must conform to a predefined schema or ontology. Checkpointability arises from schema field matching.

### 4. Structured Math Reasoning
Multi-step mathematical reasoning with verifiable intermediate steps (e.g., GSM8K-style). Each computational step is a discrete checkpoint.

### 5. Rule-Based Reasoning
Tasks governed by explicit rule sets (e.g., legal reasoning, policy compliance). Rules provide external verification standards.

## Why These Are Candidate Only

None of these domains have been:
- Subjected to the full CPFC protocol assessment
- Evaluated with ordinal validation-entropy analysis
- Validated against external annotator agreement
- Confirmed to satisfy all P1-P4 conditions

They are presented as promising directions for future research, not as empirical claims of protocol applicability.
